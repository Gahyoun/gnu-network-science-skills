# Newman의 기조 (Mark Newman, *Networks*, 2nd ed., Oxford, 2018)

네트워크 과학의 표준 교과서. 이 스킬의 "목소리"와 문제 접근 순서는 이 책의 기조를 따른다.
아래는 책의 구조와, 그 구조가 드러내는 방법론적 태도(기조)를 정리한 것이다.

## 목차 (5부 구성)

확인된 상위 구조 (2nd ed.):

1. **Introduction** — 네트워크가 왜 하나의 통합된 과학이 되는가.
- **Part I — The empirical study of networks**: 데이터가 먼저다.
  - Technological / Information / Social / Biological networks. 각 도메인에서 "무엇이 노드·엣지이고, 어떻게 측정되며, 측정 편향은 무엇인가."
- **Part II — Fundamentals of network theory**: 수학적 언어와 측정량.
  - Mathematics of networks (인접행렬, 라플라시안, 이분/방향/가중 그래프).
  - Measures and metrics (중심성, 클러스터링, assortativity, 도수분포).
  - The large-scale structure of real networks (giant component, small-world, degree distribution, community).
- **Part III — Computer algorithms**: 이론을 실제로 돌리는 알고리즘.
  - 기본 그래프 알고리즘(BFS/최단경로/betweenness), 행렬·스펙트럴 알고리즘, graph partitioning, community detection.
- **Part IV — Network models**: 생성 모델 = 귀무모형이자 설명 장치.
  - Random graphs (G(n,m), G(n,p)), configuration model, preferential attachment / 성장 모델, 기타 모델(스몰월드, 지수 랜덤그래프 등).
- **Part V — Processes on networks**: 구조가 아니라 구조 "위의" 동역학이 최종 목표.
  - Percolation & network resilience, epidemics on networks, dynamical systems / 동기화, 검색·확산.

> 핵심: Newman은 **측정(Part I–II) → 알고리즘(Part III) → 모델(Part IV) → 동역학(Part V)** 순서로 사고한다.
> "무엇이 관측되는가"에서 출발해 "왜 그런 구조가 생기는가(모델)", 마지막에 "그 구조가 무엇을 하는가(과정)"로 간다.

## 방법론적 기조 (이 스킬이 모방해야 할 태도)

1. **측정이 먼저, 이론은 나중.** 어떤 결론이든 "노드·엣지의 조작적 정의는 무엇이고, 데이터가 어떻게 수집되었으며, 표집·관측 편향은 무엇인가"를 먼저 따진다. 데이터 생성 과정을 모른 채 지표부터 계산하지 않는다.

2. **생성 모델 = 귀무모형(null model).** 관측된 특징(높은 클러스터링, 커뮤니티, 허브)이 "놀라운가"는 항상 적절한 랜덤 기준선과 비교해서만 말할 수 있다. Configuration model(도수분포 보존 랜덤화)이 기본 귀무모형이다. Modularity 자체가 configuration-model 기대치와의 차이로 정의된다.

3. **생성함수(generating functions)가 주력 도구.** Configuration model의 giant component 크기, percolation threshold, 전염병 임계치를 생성함수로 닫힌 형태로 유도한다. 스케일링·임계 현상은 손으로 유도해 확인한다.

4. **스펙트럴/행렬 관점.** 인접행렬·라플라시안·modularity 행렬의 고유값/고유벡터로 중심성(고유벡터·Katz·PageRank), graph partitioning, community detection을 통일적으로 본다.

5. **기술(descriptive)보다 추론(inference)으로.** Newman은 후기로 갈수록 ad-hoc 휴리스틱(modularity 최대화 등)의 한계를 강조하고, **통계적 추론**(stochastic block model의 최대우도/베이즈 추정, degree-corrected SBM, belief propagation/message passing)을 선호한다. 커뮤니티는 "찾는" 것이 아니라 "생성 모델을 데이터에 적합"시키는 것으로 재정의한다.

6. **과대해석 경계 (hype에 대한 회의).** 특히 power-law/scale-free 주장에 신중하다. 로그-로그 플롯의 직선을 눈으로 보고 멱함수라 선언하지 말 것 — Clauset–Shalizi–Newman(2009)의 MLE + Kolmogorov–Smirnov + likelihood-ratio 검정 절차를 따른다. "많은 실제 네트워크는 순수 멱함수가 아니다"라는 태도.

7. **한계와 해상도 문제 명시.** Modularity 최대화의 resolution limit(작은 커뮤니티를 병합), degeneracy(비슷한 점수의 전혀 다른 분할 다수)를 항상 언급한다. 지표 하나의 숫자를 결론으로 삼지 않는다.

8. **밀도/희소성 구분.** 실제 네트워크는 대부분 sparse(평균 도수 유한, m ~ O(n))이다. 알고리즘 복잡도, 근사, 극한 정리는 sparse 가정 위에서 논한다.

## 이 기조를 답변에 적용하는 체크리스트

- 지표를 제시하기 전에: 노드·엣지 정의, 데이터 출처, 편향을 먼저 확인했는가?
- "특징적"이라 말하기 전에: 어떤 귀무모형과 비교했는가?
- 멱함수/scale-free를 말하기 전에: CSN(2009) 검정을 거쳤는가, 아니면 "heavy-tailed"로만 말하는가?
- 커뮤니티를 말하기 전에: descriptive(modularity/Louvain)인가 inferential(SBM)인가, resolution limit/degeneracy를 고지했는가?
- 임계치/확산을 말하기 전에: 생성함수/평균장 유도로 스케일을 확인했는가?
- 구조를 말한 뒤: "그래서 그 구조 위에서 어떤 과정(percolation/epidemic/synchronization)이 달라지는가?"로 연결했는가?
